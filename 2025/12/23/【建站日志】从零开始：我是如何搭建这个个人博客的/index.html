【建站日志】从零开始：我是如何搭建这个个人博客的

发表于2025-12-23|更新于2025-12-24|技术随笔
|浏览量:

“装饰只是‘装修房子’，维护和写作才是‘过日子’。”

为什么要做这个博客？

在这个算法推荐横行的时代，我总觉得缺少一块真正属于自己的“自留地”。朋友圈太吵，笔记软件太封闭。我想要一个地方，可以安安静静地记录技术、生活和碎碎念，不用担心被审核，也不用在意有没有人点赞。

于是，我决定搭建这个 Hexo 博客。

我的“军火库” (技术栈)

为了把博客弄得既好看又好用，我折腾了一套目前比较流行的组合：

  • 核心框架： Hexo (生成静态网页的神器)
  • 托管平台： GitHub Pages (免费、稳定，程序员的浪漫)
  • 主题： Butterfly (功能强大，颜值抗打)
  • 写作工具： Obsidian (本地知识管理，写作体验极佳)
  • 图床方案： PicGo + GitHub (实现截图一键上传)

装修之路：给博客“注入灵魂”

刚生成的 Hexo 博客只是一个简陋的白板。为了让它变好看，我做了亿点点修改：

  1. 全站毛玻璃特效： 给所有卡片加上了透明度和圆角，让背后的星空/雪山壁纸能透出来，科技感拉满。
  2. 看板娘： 左下角请了一位二次元“看板娘”，没事还能跟她互动（虽然她有时候有点吵）。
  3. 不中断音乐： 加上了 Aplayer 播放器和 PJAX 技术，切换页面时音乐不会断，浏览体验像 App 一样丝滑。
  4. 防盗链壁纸： 这一点折腾了最久，最后通过本地存储图片解决了背景时不时变蓝的问题。

踩坑实录 (血泪史)

搭建过程并不是一帆风顺的，我踩了几个新手必踩的“深坑”，记录下来警示后人（也提醒未来的自己）。

1. 诡异的“乱码”事件

刚开始生成“标签页”和“关于页”时，网页上显示的标题全是乱码。
原因： PowerShell 命令行默认生成的文件是 GBK 编码，而网页只认 UTF-8。
教训： 永远、永远使用 VS Code 编辑文件，并确保右下角显示的是 UTF-8。

2. 消失的背景图

配置了背景图，本地能访问，但网页上一片蓝。
原因： 我把图片放在了博客根目录，但 Hexo 发布时只认 source 文件夹里的东西。
解决： 必须把图片扔进 source/img/ 文件夹，并且在配置里写绝对路径 /img/bg.jpg。

3. PowerShell 的“红字”报错

安装插件时，终端疯狂报错，提示“禁止运行脚本”。
原因： Windows 的安全策略默认太保守。
解决： 需要用管理员身份运行 set-executionpolicy remotesigned 解除封印。

现在的“极简”工作流

最开始我担心维护博客很麻烦，要敲一堆命令。但经过优化，我现在的工作流是这样的：

  1. 写作： 在 Obsidian 里舒服地写文章，配合 PicGo，截图直接 Ctrl+V 就能上传图片。
  2. 发布： 写完后，只需要运行一个我写好的 deploy.ps1 脚本。

这个脚本会自动帮我做四件事：

  • 🧹 清理缓存 (hexo clean)
  • 🔨 生成网页 (hexo generate)
  • 🌍 发布上线 (hexo deploy)
  • 💾 备份源码到 GitHub (这最重要！)

写在最后

折腾博客的过程，其实比写博客本身还有趣。从面对一堆报错的焦虑，到看着网页一点点变成自己喜欢的样子，这种成就感是无与伦比的。

现在，“房子”装修好了，接下来就是好好“过日子”——多写、多记、多分享。
